Control Flow

Make your programs smart — learn how to branch with if/else, match with switch, condense with the ternary operator, and safely combine conditions with short-circuit evaluation.

Exercise: ~15 min

Part 1 — if / else if / else

Without control flow, every program does exactly the same thing every time — not very useful. The if statement is Java's most fundamental decision-maker: it evaluates a boolean condition, and if it's true, the block inside the curly braces runs; if it's false, it's skipped entirely. Chain else if branches to check additional conditions, and add a final else to catch everything that didn't match.

Basic Structure

The condition inside the parentheses must be a boolean expression — any comparison, logical operation, or boolean variable.

int score = 72;

if (score >= 90) {
    System.out.println("A — Excellent!");
} else if (score >= 80) {
    System.out.println("B — Good job!");
} else if (score >= 70) {
    System.out.println("C — Passing");
} else if (score >= 60) {
    System.out.println("D — Needs improvement");
} else {
    System.out.println("F — Please see your teacher");
}
💡 How Java evaluates the chain

Java tests conditions top to bottom and stops at the first true one. Since we check score >= 90 first, a score of 95 only prints the "A" message — it never reaches the >= 80 check. Order matters.

Conditions Can Be Any Boolean Expression

Your condition can be as simple as a single variable or as complex as you need. Here are common patterns you'll use constantly:

int age = 15;
boolean hasTicket = true;
String weather = "sunny";

// Single boolean variable
if (hasTicket) {
    System.out.println("Welcome!");
}

// Combining conditions with && and ||
if (age >= 13 && age <= 17) {
    System.out.println("You're a teenager");
}

if (weather.equals("sunny") || weather.equals("cloudy")) {
    System.out.println("Good day for a walk");
}

// Negation with !
if (!hasTicket) {
    System.out.println("You need a ticket to enter");
}
⚠️ Always use curly braces

Java allows you to skip { } if the body is a single line, but this is a trap. Adding a second line later makes it look like it's inside the if block when it isn't. Always use curly braces — every professional Java codebase does.

⚠️ == vs .equals() for Strings

Use == to compare numbers and booleans. For Strings, always use .equals(). Writing if (name == "Alex") might not work as expected — == checks if two String variables point to the same object in memory, not whether they contain the same text.

Part 2 — switch / case

When you're comparing a single variable against many specific exact values, a switch statement is often cleaner than a long chain of if/else if. Think of it like a menu — Java jumps directly to the matching case, runs the code there, and stops when it hits break. A default case catches anything that didn't match, just like a final else.

Classic switch / case

The switch expression can be an int, char, String, or an enum (no boolean, no double). Each case must end with break — if you forget it, Java keeps running into the next case.

int day = 3;

switch (day) {
    case 1:
        System.out.println("Monday");
        break;
    case 2:
        System.out.println("Tuesday");
        break;
    case 3:
        System.out.println("Wednesday");
        break;
    case 4:
        System.out.println("Thursday");
        break;
    case 5:
        System.out.println("Friday");
        break;
    case 6:
    case 7:
        System.out.println("Weekend!");   // cases 6 AND 7 both reach this
        break;
    default:
        System.out.println("Invalid day");
}
⚠️ Fall-through: Java's biggest switch surprise

If you forget break, Java doesn't stop — it "falls through" into the next case and runs that code too, whether it matches or not. This is almost always a bug. The only intentional use is stacking cases that share the same action (like cases 6 and 7 above).

Seeing Fall-Through in Action

Run this and watch what happens when break is missing — it's one of those things you have to see once to remember forever:

int x = 1;

switch (x) {
    case 1:
        System.out.println("One");
        // no break — falls through!
    case 2:
        System.out.println("Two");
        // no break — falls through!
    case 3:
        System.out.println("Three");
        break;
    case 4:
        System.out.println("Four");
        break;
}
// Output: One  Two  Three  (all three print!)

Enhanced switch (Java 14+)

Modern Java has a cleaner switch syntax using -> (arrow). No break needed, no fall-through possible, and it can return a value directly. The code below is the classic equivalent — the commented-out block at the bottom shows what the Java 14+ arrow syntax looks like:

String day = "MONDAY";
String type;

switch (day) {
    case "MONDAY": case "TUESDAY": case "WEDNESDAY":
    case "THURSDAY": case "FRIDAY":
        type = "Weekday"; break;
    case "SATURDAY": case "SUNDAY":
        type = "Weekend"; break;
    default:
        type = "Unknown";
}

System.out.println(type); // Weekday

// Java 14+ arrow syntax (not supported in this runner — shown for reference):
// String type = switch (day) {
//     case "MONDAY", "TUESDAY", "WEDNESDAY", "THURSDAY", "FRIDAY" -> "Weekday";
//     case "SATURDAY", "SUNDAY" -> "Weekend";
//     default -> "Unknown";
// };
💡 switch vs if/else — when to use which

Use switch when checking one variable against exact values (days, menu options, single characters). Use if/else when you need ranges (>= 90), complex conditions (&&, ||), or comparing different variables.

Part 3 — The Ternary Operator

Sometimes a full if/else block feels like a lot of code just to pick between two values. The ternary operator is a compact one-line alternative. Its name comes from the fact that it takes three parts: a condition, a value if true, and a value if false.

Syntax and Usage

The format is: condition ? valueIfTrue : valueIfFalse. The whole expression evaluates to one of the two values, so you can assign it directly to a variable or use it inside System.out.println.

int age = 16;

// Ternary — one line
String status = age >= 18 ? "Adult" : "Minor";
System.out.println(status);  // Minor

// Equivalent if/else — four lines
String status2;
if (age >= 18) {
    status2 = "Adult";
} else {
    status2 = "Minor";
}

// Ternary directly inside a print
int score = 85;
System.out.println("Result: " + (score >= 60 ? "Pass" : "Fail"));

// Ternary with maths
int a = 7, b = 12;
int max = a > b ? a : b;
System.out.println("Max: " + max);  // 12
⚠️ Don't nest ternaries

You can nest one ternary inside another, but it quickly becomes unreadable. As a rule: if your ternary doesn't fit on one clean line, use if/else instead. Code is read far more often than it's written — clarity wins.

Part 4 — Short-Circuit Evaluation

You already know && (AND) and || (OR) from Lesson 1. There's a deeper behaviour worth understanding: Java is lazy about evaluating them. The moment it knows the final answer, it stops and doesn't evaluate the rest. This is called short-circuit evaluation, and it's not just an optimisation — it prevents real errors.

How Short-Circuit Works

With &&: if the left side is false, the whole expression is false no matter what, so Java skips the right side. With ||: if the left side is true, the whole expression is true, so Java skips the right side.

// && short-circuit: stops if left side is false
int x = 0;
if (x != 0 && 10 / x > 2) {       // 10/x would crash if x is 0
    System.out.println("Big");     // this line is safe — x != 0 is false,
}                                  // so Java never evaluates 10 / x

// || short-circuit: stops if left side is true
boolean isAdmin = true;
if (isAdmin || checkDatabasePermission()) {  // checkDatabasePermission() is
    System.out.println("Access granted");    // never called — isAdmin is true
}

// Practical example: null safety
String name = null;
if (name != null && name.length() > 0) {
    System.out.println("Name: " + name);   // safe — null check fires first
}
// Without short-circuit, name.length() on null would throw NullPointerException
💡 The golden rule for null safety

Always put the null check first with &&. name != null && name.length() > 0 is safe. Flipping the order to name.length() > 0 && name != null will crash with a NullPointerException because Java evaluates left-to-right and tries name.length() before discovering name is null.

De Morgan's Laws

Two rules that help you simplify or rewrite complex boolean conditions. You'll use these when debugging or refactoring conditions:

// De Morgan's Laws — these pairs are equivalent:

// NOT (A AND B)  ==  (NOT A) OR  (NOT B)
!(a && b)  ==  (!a || !b)

// NOT (A OR  B)  ==  (NOT A) AND (NOT B)
!(a || b)  ==  (!a && !b)

// Real example — "not a teenager":
int age = 25;
if (!(age >= 13 && age <= 17)) {
    System.out.println("Not a teenager");
}
// Equivalent (De Morgan applied):
if (age < 13 || age > 17) {
    System.out.println("Not a teenager");
}
// The second form is usually clearer to read

Exercise — 15 Minutes
⌨️ Coding Challenge ⏱ ~15 min

Task 1 — FizzBuzz
The classic programmer interview puzzle. Write a program that checks a single integer. If it's divisible by 3, print "Fizz". If divisible by 5, print "Buzz". If divisible by both 3 and 5, print "FizzBuzz". Otherwise, print the number itself. Test with: 15, 9, 10, 7.

Hint: check divisibility with the modulo operator — n % 3 == 0 is true when n is divisible by 3.

The order of your conditions matters. Check n % 3 == 0 && n % 5 == 0 (both) first, then check each separately. If you check "divisible by 3" first, a number like 15 will print "Fizz" and never reach the "FizzBuzz" check.

Task 2 — Season Finder
Write a program using a switch statement that takes a month number (1–12) and prints the season. Use: Dec/Jan/Feb → Winter, Mar/Apr/May → Spring, Jun/Jul/Aug → Summer, Sep/Oct/Nov → Autumn. Test with months 1, 4, 8, 11. What happens if you enter 13?

Stack multiple cases that share the same result: case 12: case 1: case 2: System.out.println("Winter"); break;. Add a default case to handle invalid inputs like 13.

Task 3 — Leap Year Checker
A year is a leap year if: it is divisible by 4 AND (not divisible by 100 OR divisible by 400). Write a program that checks a year. Test with: 2000 (leap), 1900 (not leap), 2024 (leap), 2023 (not leap). Try writing the condition as a single if expression, then verify with a ternary.

The condition in Java: year % 4 == 0 && (year % 100 != 0 || year % 400 == 0). Parentheses are essential here because && has higher precedence than ||. Without them, the logic is wrong.
