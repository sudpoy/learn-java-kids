Part 1 — How Java Works

The JVM: Write Once, Run Anywhere

Most programming languages compile directly to machine code for a specific computer (Windows code won't run on Mac). Java takes a different approach using the Java Virtual Machine (JVM).

When you write Java, two steps happen:

  • Compile — javac turns your .java file into .class bytecode (not machine code — something in between)
  • Run — the JVM reads that bytecode and translates it for whatever machine you're on
💡 Why this matters

The same .class file runs on Windows, Mac, Linux, Android — without recompiling. That's why Java powers Android apps, Minecraft, and millions of enterprise servers simultaneously.

Your First Java Program

Hello.java
public class Hello {                          // class name must match filename
    public static void main(String[] args) {  // entry point — Java starts here
        System.out.println("Hello, World!");  // print to console
        System.out.print("No newline here");  // print WITHOUT newline
        System.out.printf("I am %d years old%n", 13); // formatted print
    }
}
⚠️ Common mistake

The class name (Hello) must exactly match the filename (Hello.java) — case matters. This trips up everyone at first.

Part 2 — The 8 Primitive Types

Java's Type System

Java is statically typed — every variable must declare its type and it never changes. This catches mistakes at compile time before your code even runs.

💡 String is NOT a primitive

String is a capital-S class (an object), not a primitive. It gets special treatment in Java — you can declare it like a primitive, but it has methods and behaves differently.

Part 3 — Variables & Constants

Declaring and Assigning Variables

// Declaration + assignment in one line (most common)
int score = 100;
double gpa = 3.85;
boolean isPassing = true;
char initial = 'J';
String name = "Alex";

// Declare first, assign later
int level;
level = 5;          // must assign before use or compiler will error

// Multiple declarations of same type
int x = 10, y = 20, z = 30;

// Constants — use final, ALL_CAPS by convention
final double PI = 3.14159265;
final int MAX_STUDENTS = 30;
// PI = 3.0;  // ❌ COMPILE ERROR — can't reassign a final variable

// var (Java 10+) — type inferred from value
var message = "Hello!";  // Java infers: String
var count = 42;           // Java infers: int

Naming Rules

Variable names: start with a letter, $, or _; no spaces; can't be reserved words (int, class, etc.).

Java convention: camelCase for variables and methods (playerScore, firstName), PascalCase for classes (Student, BankAccount).

Part 4 — Operators

Arithmetic Operators

int a = 17, b = 5;

System.out.println(a + b);   // 22  addition
System.out.println(a - b);   // 12  subtraction
System.out.println(a * b);   // 85  multiplication
System.out.println(a / b);   // 3   integer division — decimal truncated!
System.out.println(a % b);   // 2   modulo (remainder)

// To get a decimal result, cast first:
System.out.println((double) a / b);  // 3.4

// Shorthand operators
a += 3;   // a = a + 3 → 20
a -= 1;   // a = a - 1 → 19
a *= 2;   // a = a * 2 → 38
a /= 4;   // a = a / 4 → 9
a %= 4;   // a = a % 4 → 1

// Increment / Decrement
int x = 5;
x++;      // x = 6  (post-increment)
++x;      // x = 7  (pre-increment)
x--;      // x = 6  (post-decrement)

// Difference matters in expressions:
int y = 5;
System.out.println(y++);  // prints 5, THEN increments
System.out.println(++y);  // increments FIRST, then prints 7

Comparison & Logical Operators

// Comparison — always produces boolean
System.out.println(5 == 5);   // true   equal to
System.out.println(5 != 3);   // true   not equal
System.out.println(7 > 5);    // true   greater than
System.out.println(3 >= 3);   // true   greater than or equal
System.out.println(2 < 1);    // false  less than

// Logical — combine booleans
boolean a = true, b = false;
System.out.println(a && b);   // false  AND — both must be true
System.out.println(a || b);   // true   OR  — at least one must be true
System.out.println(!a);       // false  NOT — flips the value

// Short-circuit evaluation
// If left side of && is false, right side is never checked
// If left side of || is true, right side is never checked
int x = 0;
boolean r = (x != 0) && (10 / x > 1);  // safe — never divides by 0

String Concatenation Trap

// + with Strings: everything to its right becomes text
System.out.println("Score: " + 10 + 5);   // "Score: 105"  ← NOT 15!
System.out.println("Score: " + (10 + 5)); // "Score: 15"   ← correct

// Any type + String = String
int n = 42;
String s = "Answer: " + n;   // "Answer: 42"

// Concatenating booleans and chars
System.out.println("Flag: " + true);   // "Flag: true"
System.out.println("Char: " + 'A');    // "Char: A"
⚠️ The #1 string concatenation gotcha

"result: " + 1 + 2 gives "result: 12". The + is left-to-right: first it joins the string and 1 (making "result: 1"), then joins that with 2. Always use parentheses when mixing arithmetic and strings.

Part 5 — Type Casting

Widening vs Narrowing Conversion

Widening (safe, automatic): going from a smaller type to a larger one — no data is lost.
Narrowing (manual cast required): going from larger to smaller — data can be lost. 

// WIDENING — automatic (byte → short → int → long → float → double)
int i = 42;
double d = i;          // automatic: 42 → 42.0
long l = i;            // automatic: 42 → 42L
System.out.println(d); // 42.0

// NARROWING — requires explicit cast with (type)
double pi = 3.14159;
int truncated = (int) pi;    // 3 — decimal part is DROPPED (not rounded!)
System.out.println(truncated); // 3

// Integer division gotcha
int a = 7, b = 2;
System.out.println(a / b);            // 3  ← wrong if you want 3.5
System.out.println((double) a / b);   // 3.5 ✓  cast a before dividing

// char ↔ int (char is stored as its Unicode number)
char c = 'A';
int code = c;                         // widening: 65
System.out.println(code);             // 65

int ascii = 66;
char letter = (char) ascii;           // narrowing
System.out.println(letter);           // B

// 'A'=65, 'Z'=90, 'a'=97, 'z'=122, '0'=48
// So you can do arithmetic on chars:
System.out.println((char)('a' + 3));  // d

⌨️ Coding Challenge ⏱ ~15 min

Task 1 — Temperature Converter
Write a program that stores a temperature in Celsius as a double, then calculates and prints the Fahrenheit equivalent. Formula: F = (C × 9/5) + 32. Watch out for integer division! Try: 0°C, 100°C, 37°C (body temperature).

The formula involves 9/5 — if both are int, Java gives 1 (integer division). Use 9.0/5 or cast: (double)9/5. Your variable should be double celsius = 100; and compute double fahrenheit = (celsius * 9.0/5) + 32;

Task 2 — Char Cipher
Store the character 'A' in a char variable. Print its integer (ASCII) value. Then print the character that is 3 positions after it in the alphabet. Now do the same for 'm'. What do you get?

Cast char to int to see its value: (int)'A' → 65. Add 3 to get 68, cast back to char: (char)('A' + 3) → 'D'. For lowercase 'm': (char)('m' + 3) → 'p'.

Task 3 — Type Detective
Without running the code, predict the output of each line. Then verify:

System.out.println(10 / 3);
System.out.println(10.0 / 3);
System.out.println((int) 9.9);
System.out.println(2 + 3 + " hello");
System.out.println("hello " + 2 + 3);
System.out.println('A' + 1);

10 / 3 → 3 (int division)
10.0 / 3 → 3.3333... (double division)
(int) 9.9 → 9 (truncated)
2 + 3 + " hello" → "5 hello" (math first, then string)
"hello " + 2 + 3 → "hello 23" (string concat left-to-right)
'A' + 1 → 66 (char treated as int in arithmetic!)
